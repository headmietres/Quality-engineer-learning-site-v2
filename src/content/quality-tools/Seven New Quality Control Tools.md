# QC 新七大手法 — 详细解读与制造业实战案例

## 概述

QC新七大手法（又称【管理与策划七大工具】起源于1970年代日本，由纳谷嘉信博士等人归纳提出。

> 与旧七大手法的区别：旧七大手法侧重于数据分析与品质控制，新七大手法主要是用于语言资料的整理，计划的策定与问题的系统性解析

---

## 一、关联图（Relations Diagram）

### 1.1核心概念

将**复杂的因果**关系以逻辑箭头进行连接，找出多个因素之间的相互关联，进而厘清问题的核心或者关键项目。

### 1.2 适用场景

- 问题涉及多种因素、因果关系交叉缠绕
- 需要同时处理多种原因与结果的关系
- 团队讨论中需要统一认知

### 1.3 作法步骤

1.**列出所有相关因素**：将认为与事件相关的所有原因、结果、现象写在卡片上\
2.**分析因果关系**：用箭头从「原因」指向「結果」（A → B 表示 A 导致 B）\
3.**反复检查箭头方向**：确认每一条箭头的逻辑是否正确\
4.**找出关键因素**：
- 箭头只出不进 → 根本原因（源头因素）
- 箭头只进不出 → 最终結果（需要解決的问题）
- 进出箭头都多 → 中間的关键枢纽

### 1.4 实践案例：汽車零件冲压厂——冲压件毛边不良率偏高的原因分析
#### 背景：
某汽车冲压厂的主要产品为车身钣金件。近三个月以来，冲压件的毛边不良率从原本的 1.2% 上升到 4.8%，严重影响后段焊接品质与交期，品管部门尝试使用柏拉图进行分析，发现毛边问题涉及多种原因交叉影响，决定使用关联图进行系统性分析。
#### 步骤一:收集可能因素（跨部门头脑风暴）
由冲压课，品保课，模具课组成小组，经过讨论列出以下几种可能：
| 编号 | 因素 |
|------|------|
| A | 冲压模具间隙过大 |
| B | 模具刃口磨损 |
| C | 模具维护保养周期过长 |
| D | 冲压次数超过模具寿命|
| E | 材料硬度波动（钢材材质不稳定）|
| F | 润滑油膜不均匀|
| G | 润滑油供给系统故障 |
| H | 操作人员未按照标准作业 |
| I | 新进员工培训不到位 |
| J | 模具更换标准不明确 |
| K | 毛边不良率上升 |
| L | 后段焊接品质异常 |
| M | 模具更换标准不明确 |

#### 步骤二：分析因果关系

```
A（模具间隙过大）─────→ K（毛边不良率上升）
B（刃口磨损）────────→ A（模具间隙过大）
C（维护保养周期过长）──→ B（刃口磨损）
D（冲压次数超过寿命）──→ B（刃口磨损）
D ────────────────────→ C（维护保养周期过长）
E（材料硬度波动）─────→ A（模具间隙过大）
F（润滑油膜不均匀）───→ B（刃口磨损）
G（润滑油系统故障）───→ F（润滑油膜不均匀）
H（冲压速度过快）─────→ F（润滑油膜不均匀）
I（未按标准作业）─────→ H（冲压速度过快）
J（新进人员训练不足）──→ I（未按标准作业）
M（模具更换标准不明确）→ C（维护保养周期过长）
M ────────────────────→ D（冲压次数超过寿命）
K（毛边不良率上升）───→ L（后段焊接品质异常）
```
#### 步骤三：分析箭头进出，找出关键因素

| 因素 | 分类 | 说明 |
|------|------|------|
| J 新进人员训练不足 | 根本原因 | 箭头只出不进 |
| G 润滑油系统故障 | 根本原因 | 箭头只出不进 |
| E 材料硬度波动 | 根本原因 | 箭头只出不进 |
| M 模具更换标准不明确 | 根本原因 | 箭头只出不进 |
| B 刃口磨损 | 关键枢纽 | 进出箭头都多（4进1出） |
| L 后段焊接质量异常 | 最终原因 | 箭头只进不出 |

### 结论与对策
| 因素 | 分类 | 对策 |
|------|------|------|
| J 新进人员训练不足 | 根本原因 | 强化新人教育训练制度，增加实作考核 |
| G 润滑油系统故障 | 根本原因 | 每日点检润滑系统，纳入 TPM 保养项目 |
| E 材料硬度波动 | 根本原因 | 与钢材供货商协商质量标准，进料增加硬度检验 |
| M 模具更换标准不明确 | 根本原因 | 制定明确的模具寿命管理标准，规定冲压次数上限 |
| B 刃口磨损 | 关键枢纽 | 缩短模具维护周期，增加刃口自主检查频率 |

## 二、亲和图（Affinity Diagram / KJ 法）

  将**大量零散的语言数据**（意见、想法、事实）按照其**内在的亲近性**（相似性）自然归类分群，从中发现新的构想或洞察。由日本人类学者川喜田二郎（KJ）创始。

**适用场景**
- 头脑风暴后需要整理的想法
- 问题模糊、方向不明确，需要从混沌中理出头绪
- 需要「归纳式」地从具体事实中发现抽象概念

> 亲和图 vs 关联图：亲和图的核心动作是分类归组（归纳），适合问题模糊、需要探索的阶段；关联图的核心动作是分析因果（演绎），适合问题明确、需要追根究底的阶段。
### 实战案例：电子组装厂 —— 客户满意度下降原因的探索性调查
#### 背景：
某 PCB 电子组装代工厂（EMS），客户满意度从 87 分下降到 72 分，但客户回馈意见非常零散。品保部决定用亲和图进行系统性归类。
#### 步骤一：搜集语言数据（列举 20 条）

| 序号 | 客户意见 |
|------|------|
| 1 | 「上次交货的 SOP-220 板子有 3 片空焊」 |
| 2 | 「交货日期经常延迟 1-2 天」 |
| 3 | 「回复 E-Mail 的速度太慢」 |
| 4 | 「包装箱内没有放干燥剂，板子受潮」 |
| 5 | 「不良品退货的处理流程不清楚」 |
| 6 | 「产线换线后的第一批质量不稳定」 |
| 7 | 「出货检验报告的格式每次不一样」 |
| 8 | 「要求提供制程能力报告，拖了两周才收到」 |
| 9 | 「IC 组件脚弯曲的问题一直没改善」 |
| 10 | 「物流司机送错厂区」 |
| 11 | 「报价回复速度比竞争对手慢」 |
| 12 | 「新产品试产的沟通窗口不固定」 |
| 13 | 「同一产品不同批次的焊锡质量差异大」 |
| 14 | 「产品卷标上的批号经常印错」 |
| 15 | 「质量异常发生时，8D 报告要等一个月」 |
| 16 | 「要求 CAR 回复，对方说要内部确认后就没下文」 |
| 17 | 「外箱标示与实际数量不符」 |
| 18 | 「ICT 测试数据没有随货提供」 |
| 19 | 「接洽窗口换了三次，每次都重新解释需求」 |
| 20 | 「交货时混入了上一批次的库存品」 |

#### 步骤二：反复分组（3 轮归纳）

**第一轮 → 小群组：**

| 小群组标题 | 包含卡片 |
|------------|----------|
| 焊接质量问题 | 卡片 1、9、13 |
| 交期延迟 | 卡片 2 |
| 响应速度慢 | 卡片 3、8、11、15、16 |
| 包装与标示问题 | 卡片 4、14、17、20 |
| 文件与数据管理 | 卡片 7、18 |
| 沟通窗口不固定 | 卡片 12、19 |
| 退货流程不清晰 | 卡片 5 |
| 换线质量波动 | 卡片 6 |
| 物流错误 | 卡片 10 |

**第二轮 → 大分类：**

| 大分类 | 包含的小群组 |
|--------|--------------|
| 产品质量问题 | 焊接质量问题、换线质量波动 |
| 响应与沟通问题 | 响应速度慢、沟通窗口不固定 |
| 出货与物流问题 | 交期延迟、包装与标示问题、物流错误 |
| 文件与流程问题 | 文件与数据管理、退货流程不清晰 |

#### 步骤三：亲和图成果与核心发现
```
┌──────────────────────────────────────────────────────────────┐
│              客户满意度下降原因亲和图                           │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌─ 🔴 产品质量问题 ─────────────────────────┐              │
│  │  • 焊接质量问题（空焊、脚弯曲、批次差异）       │              │
│  │  • 换线品质波动                              │              │
│  └────────────────────────────────────────────┘              │
│                         ↓                                    │
│  ┌─ 🟡 响应与沟通问题 ────────────────────────┐              │
│  │  • 响应速度慢（E-Mail、报价、8D报告）           │              │
│  │  • 沟通窗口频繁更换                           │              │
│  └────────────────────────────────────────────┘              │
│                         ↓                                    │
│  ┌─ 🔵 出货与物流问题 ────────────────────────┐              │
│  │  • 交期延迟                                   │              │
│  │  • 包装受潮、标示错误、数量不符                 │              │
│  │  • 物流送错厂区、混入旧批库存                   │              │
│  └────────────────────────────────────────────┘              │
```

 > 核心发现
占比最高且影响最大的是「响应与沟通问题」和「出货与物流问题」。这两类问题本质上反映了客户服务体系的系统性缺陷，而非单点问题。

## 三、系统图（Systematic Diagram / Tree Diagram）

将一个**目的（目标）**层层展开为具体的**手段与方法**，形成树状结构。又称「树图」或「目的别展开图」。

### 类型

| 类型 | 说明 |
|------|------|
| **由上而下型** | 从最高层目的出发，逐层问「用什么手段可以达成？」 |
| **由下而上型** | 从具体现象出发，逐层问「为什么会这样？」 |

###  实战案例：食品包装厂 —— 降低产品异物混入不良率

> **背景：** 某食品包装工厂生产真空包装即食米饭，异物混入不良率达 **0.3%**（客户要求 ≤ 0.05%）。质量部用**系统图**逐级展开控制手段。


```
【目的】异物混入不良率 ≤ 0.05%
│
├─── ① 杜绝异物来源
│    ├─── ①-1 毛发控制
│    │    ├─── a. 全员穿戴头套＋胡须套＋发网（三层防护）
│    │    ├─── b. 入厂风淋通道升级为双门互锁式
│    │    └─── c. 车间内增设黏尘滚筒（每 2 小时全员滚一次）
│    │
│    ├─── ①-2 塑料碎片控制
│    │    ├─── a. 更换包装膜供货商（改用 FDA 认证薄膜）
│    │    ├─── b. 封口机刀具更换周期从每月改为每周
│    │    └─── c. 进料检验增加膜材厚度均匀度检测
│    │
│    └─── ①-3 金属屑控制
│         ├─── a. 所有金属设备零件改为不锈钢材质
│         ├─── b. 设备保养时禁止在产线上方进行研磨作业
│         └─── c. 产线入口增设金属检测机（灵敏度 Fe ≥ 1.5mm）
│
├─── ② 防止异物混入产品
│    ├─── ②-1 物理隔离
│    │    ├─── a. 产线上方加装透明亚克力防护罩
│    │    ├─── b. 原料区与包装区之间设置缓冲区
│    │    └─── c. 作业台面全面改为一体成型不锈钢
│    │
│    ├─── ②-2 环境控制
│    │    ├─── a. 包装区洁净度提升至 Class 100,000
│    │    ├─── b. 空调出风口加装 HEPA 过滤网
│    │    └─── c. 每日环境落尘量监测（≤ 3 个/板/小时）
│    │
│    └─── ②-3 人员管理
│         ├─── a. 进入包装区前强制洗手＋酒精消毒
│         ├─── b. 禁止携带个人物品进入车间
│         └─── c. 每日上岗前服装穿戴检查（班长确认）
│
└─── ③ 拦截已混入的异物
     ├─── ③-1 在线检测
     │    ├─── a. 封口前增设 X-ray 异物检测机
     │    ├─── b. 封口后增设金属检测机（二次确认）
     │    └─── c. 成品全数通过 CCD 视觉检测
     │
     └─── ③-2 抽检强化
          ├─── a. 出货抽检比例从 AQL 1.0 提升至 AQL 0.25
          └─── b. 每批保留样品进行拆封检查

```
#### 可行性评估

| 手段 | 投资 | 难度 | 效果 | 优先级 |
|------|------|------|------|------|
| ①-1-a 三层头部防护 | 低 | 低 | 高 | ★★★ 立即 |
| ②-3-b 禁带个人物品 | 无 | 低 | 中 | ★★★ 立即 |
| ①-3-c 金属检测机 | 高 | 中 | 高 | ★★★ 一个月内 |
| ③-1-a X-ray 检测机 | 很高 | 高 | 很高 | ★★ 三个月内 |
>  执行结果
第一阶段（立即低成本措施）：异物不良率从 0.30% 降至 0.12%
第二阶段（导入检测设备）：降至 0.03% ✅ 达到客户要求 ≤ 0.05%

## 四、矩阵图（Matrix Diagram）

将**两组或多组要素**以行和列排列，分析各要素之间的**对应关系**，用符号标示关联的强度。

### 常见类型

| 类型 | 结构 | 适用场景 |
|------|------|----------|
| **L 型** | 行 × 列 | 最基本，A 因素 vs B 因素 |
| **T 型** | A × B 和 A × C 共享 A | 一组同时对应另两组 |
| **Y 型** | 三组两两对应 | 三方关系分析 |
| **X 型** | 四组交叉对应 | 复杂多方关系 |
| **C 型** | 三维立体 | 三组同时对应 |

### 🏭 实战案例：半导体封装厂 —— 质量机能展开（QFD）

> **背景：** 某 IC 封装测试厂导入 QFN 封装产品，客户提出 5 项质量需求，用**矩阵图（L型）**分析客户需求与工程特性的对应关系。

**客户需求：**

| 代号 | 客户需求 |
|------|----------|
| `R1` | 焊点强度要足够（可靠性） |
| `R2` | 外观不能有溢胶 |
| `R3` | 封装尺寸精度要高 |
| `R4` | 散热性能要好 |
| `R5` | 湿气敏感度等级要达到 MSL-3 |

**L 型矩阵图：**

| | E1 打线拉力 | E2 模封压力/温度 | E3 溢胶控制 | E4 LF平面度 | E5 导热胶厚度 | E6 烘烤条件 | E7 湿度管控 |
|---|---|---|---|---|---|---|---|
| **R1 焊点强度** | ◎ | ○ | | △ | | ○ | |
| **R2 外观无溢胶** | | ◎ | ◎ | | | ○ | △ |
| **R3 尺寸精度** | | ○ | △ | ◎ | ○ | ○ | |
| **R4 散热性能** | △ | | | | ◎ | ○ | |
| **R5 MSL-3** | | ○ | | | | ◎ | ◎ |

◎＝强关联　○＝有关联　△＝弱关联

#### 分析结果

| 工程特性 | ◎ 个数 | 分析 |
|----------|--------|------|
| **E6 烘烤条件** | 3 个 | 最高优先管控 影响最多客户需求 |
| **E2 模封压力/温度** | 1◎+3○ | 第二优先 影响面广 |
| E3 溢胶控制 | 2 个 | 对外观质量影响直接 |
| E1 打线拉力 | 1 个 | 主要影响焊点可靠性 |
| E5 导热胶厚度 | 1◎+1○ | 散热的关键因素 |

**⚙️ 管理决策**

- **管制特性 CC** **E6 烘烤条件** — SPC 实时监控，Cpk ≥ 1.67
- **重要特性 SC** **E2 模封参数** — 每班首件确认

## 五、矩阵数据分析法（Matrix Data Analysis）

将矩阵交叉点的关系**量化为数据**，透过统计方法进行解析。这是七大手法中**唯一需要数值计算**的方法。

> **注意：** 这是七大手法中最复杂的工具，通常需要统计软件支持。实务中若只想做优先排序，可简化使用**加权评分法**。

### 🏭 实战案例：家电制造厂 —— 选择最佳供货商的量化评估

> **背景：** 某家电制造厂需为新产品塑料外壳选择注塑零件供货商，5 家候选（S1~S5），根据 6 项指标进行客观排序。

#### 步骤一：原始数据

| 评估指标 | 权重 | S1 | S2 | S3 | S4 | S5 |
|----------|------|----|----|----|----|----|
| 质量合格率 (%) | 30% | 96.5 | 99.2 | 94.8 | 97.1 | 98.0 |
| 交期达标率 (%) | 20% | 92.0 | 88.5 | 97.3 | 95.0 | 90.2 |
| 单价竞争力 (元/件) | 20% | 12.5 | 14.8 | 11.2 | 13.0 | 12.0 |

## 六、过程决策程序图（PDPC）

在计划执行前，**预先推演**各种可能发生的问题，并针对每个问题制定**对策方案**，确保计划能顺利达成目标。

###  实战案例：自行车零件厂 —— 碳纤维轮圈量产导入的风险预防

> **背景：** 某自行车轮圈制造厂即将导入碳纤维轮圈量产，3 个月后交首批 500 组。由于是全新制程，项目经理用 **PDPC** 预先推演风险。

```text
【目标】3 个月后碳纤维轮圈顺利量产（首批 500 组合格出货）

 步骤1 │ 原材料采购（第 1-4 周）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ● 问题 1-1：碳纤维预浸料供货商交期延迟
    └─→ 对策：同时向 2 家供货商下单，备料量增加 20%

  ● 问题 1-2：进料检验发现预浸料树脂含量超标
    └─→ 对策：与供货商签订质量协议，附出厂检验报告
        └─→ 备案：紧急联系日本备选供货商（交期 +2 周）

  ● 问题 1-3：铝合金花鼓零件缺料
    └─→ 对策：提前 2 周下单，确认库存安全量

                        ↓

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

    └─→ 对策：调整固化温度（135°C → 145°C，延长保温）
        └─→ 备案：改用高温型预浸料（+成本 15%）

  ● 问题 3-2：动平衡测试不合格（跳动量 > 0.3mm）

  ● 问题 5-1：首批合格率低于 80%
    └─→ 对策：目标从 500 组调整为 300 组
        └─→ 备案：与客户协商分批交货（300 + 200）
```

## 七、箭条图（Arrow Diagram / Network Diagram）

将项目的**所有作业**用箭头表示，按先后顺序连接成网络图，计算**关键路径**，管控进度。本质上就是 **PERT/CPM** 的图形化表达。

### 基本要素

| 要素 | 说明 |
|------|------|
| **节点（Node）** | ○ 圈圈，代表作业的开始或结束（里程碑） |
| **箭头（Activity）** | → 带方向的线，代表一个作业及所需时间 |
| **虚箭头（Dummy）** | ⤻ 虚线，表示虚拟作业（零时间），仅表达逻辑依赖 |
