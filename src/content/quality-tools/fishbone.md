# 鱼骨图：系统性地分析问题根因

---

## 1. 什么是鱼骨图？

**鱼骨图 (Fishbone Diagram)**，又称**因果图 (Cause-and-Effect Diagram)** 或**石川图 (Ishikawa Diagram)**，是由日本质量管理专家石川馨于20世纪50年代提出的一种问题分析工具。它通过结构化的图形方式，将问题的可能原因按照类别层层展开，帮助团队全面、系统地寻找根本原因，而不仅仅是停留在表面现象。

> **核心价值**：用一个团队的集体智慧，把“为什么会发生这个问题”的所有可能性都摆到纸面上，然后再用数据和事实去验证，最终锁定真正的根因。

鱼骨图广泛应用于质量改进、故障分析、流程优化、项目管理等场景，常与**5 Why 分析法**、**柏拉图 (Pareto)**、**头脑风暴** 结合使用，构成系统化问题解决的基石。

---

## 2. 鱼骨图的核心结构

一张标准的鱼骨图包含以下部分：

- **鱼头**（右侧或左侧，取决于方向）：清晰陈述的**问题或效果**。
- **主骨**：一条水平指向鱼头的粗线。
- **大骨**（分类主线）：从主骨分出的数个主要类别，通常用方框标示。
- **中骨、小骨、孙骨**：逐级细化，将模糊的原因一步步分解为可操作的潜在因子。

---

## 3. 鱼骨图的常见类型

| 类型           | 目的                           | 鱼头写法                                 |
| :------------- | :----------------------------- | :--------------------------------------- |
| **原因追求型** | 对已发生的问题，找出其根本原因 | “为什么……”（如：为什么产品表面有划痕？） |
| **对策追求型** | 对欲达成的目标，思考实现的方法 | “如何……”（如：如何将装配效率提升20%？）  |

在质量改进和问题解决中，绝大多数情况使用**原因追求型**。本文后续内容也以此为主线。

---

## 4. 鱼骨图的分析维度 (5M1E)

为确保分析能够全面覆盖，通常采用**5M1E**作为大骨的分类框架，可以根据实际情况增减：

| 类别                 | 英文                                     | 涵盖范围                               |
| :------------------- | :--------------------------------------- | :------------------------------------- |
| **人 (Man)**         | 操作者、技能、经验、身体状态、情绪、培训 | 操作失误、疲劳、责任心、沟通不足       |
| **机 (Machine)**     | 设备、工具、工装、夹具、模具             | 精度不够、磨损、异常停机、参数偏移     |
| **料 (Material)**    | 原材料、辅料、零部件、信息输入           | 材质不良、批次差异、存储变质           |
| **法 (Method)**      | 工艺方法、操作规程、标准、管理流程       | 工艺不合理、SOP不明确、参数设置错误    |
| **环 (Environment)** | 工作环境、温湿度、照明、噪音、5S         | 温度过高、灰尘、振动、照明不足         |
| **测 (Measurement)** | 测量系统、检测方法、仪器校准             | 量具不准、测量方法不一致、抽样计划缺陷 |

## 5. 绘制鱼骨图的实战步骤

### 步骤一：清晰定义问题
将问题写成简洁具体的陈述，放置在鱼头位置。问题描述必须客观、可量化，切忌笼统。

- ❌ 糟糕的写法：零件质量不好
- ✔ 好的写法：2025年3月以来，A型号轴的外径超差率从0.3%上升至2.1%

### 步骤二：确定大骨类别
根据问题的性质选择合适的大骨框架，通常用**5M1E**。将其写在主骨两侧的骨端，用方框标出。

### 步骤三：头脑风暴，填充原因
召开跨职能小组会议，以“**为什么会发生？**”为核心追问，将可能的原因逐层填入对应的大骨下。

- 讨论一个类别时，可以顺着大骨往下问：“**是什么导致了此类别的问题？**”
- 将原因写在对应的中骨上，然后再问：“**是什么导致了这个中骨原因？**” 由此扩展出小骨、孙骨，直至找到最末端的可行动因子。
- 切忌在头脑风暴阶段急于否定他人意见，先求量再求质。

### 步骤四：检查与整理
完成初稿后，对照问题反复推敲：
- 是否有遗漏的类别或因素？
- 原因与结果之间是否存在直接逻辑关系？
- 是否存在重复或可合并的条目？

### 步骤五：识别根本原因（根因验证）
鱼骨图找出的只是**可能原因**，不能直接等同于**根本原因**。需要进一步通过数据、实验或现场观察来验证。常用的验证手段包括：
- 现场确认（去现场看实物、问操作者）
- 对比分析（好件 vs. 坏件的工艺参数差异）
- 抽样检查或临时加严控制看效果
- 配合**5 Why 分析法**深入提问

将已验证的关键根因在鱼骨图上用特殊符号（如圆圈、星号）标记出来，作为后续改进措施的重点输入。

## 6. 应用案例：产品表面划伤问题

在收集的 60 余条可能原因中，小组通过现场数据验证发现两个最主要的根因：
- **传送带金属挡边磨损导致出料时接触划伤**（已观察到磨损沟槽且划痕位置完全吻合）。
- **周转工序未增加隔层保护，面板之间摩擦**（统计显示周转后划伤增加量占76%）。

针对这两项根因采取了设备修复和增加专用保护隔板措施后，划伤不良率下降了89%。

## 7. 根因验证与下一步行动

鱼骨图不是终点，而是中间站。完成鱼骨图后，应当：

1. **圈定疑点原因**：根据团队经验、频次、严重性初步选出5~8个最可能的原因。
2. **制定验证计划**：采用“现地现物”、数据对比、DOE试验等方式逐一确认。
3. **输出根因**：确认为根本原因后，进入纠正预防措施步骤（如8D报告中的D4~D5）。
4. **衔接其他工具**：
   - 使用 **柏拉图 (Pareto)** 对原因发生频率排序，优先解决主要矛盾。
   - 使用 **5 Why 分析法** 对每一个中骨/小骨持续深挖，直至找到系统性根因。
   - 将根因及措施更新至 **FMEA** 中，形成知识沉淀。

## 8. 鱼骨图的优缺点

### 优点
- **结构化与全面性**：引导团队从多个维度思考，避免片面归因。
- **高度可视化**：一张图呈现复杂问题的因果网络，便于沟通和达成共识。
- **鼓励参与**：头脑风暴过程激发团队智慧，尤其适合跨部门协作。
- **简单易用**：只需要白板、笔和便利贴，无需复杂软件。

### 缺点
- **依赖经验**：分析质量取决于参与者对过程的了解程度，可能漏掉未知因素。
- **难以量化**：各原因之间的交互作用、贡献权重无法直接从图中读出。
- **需要验证**：若缺少后续的数据验证，容易变成纸上谈兵，甚至误导决策。
- **图可能过大**：若问题过于复杂，鱼骨图会庞大无比，需拆分子问题分别分析。

---

## 9. 常见误区与注意事项

| 误区                         | 正确做法                                                     |
| :--------------------------- | :----------------------------------------------------------- |
| 问题描述太宽泛，如“质量差”   | 量化指标、限定时间、定义现象                                 |
| 把原因写成现象或抱怨         | 每个末端原因都应该是可执行、可检查的行动因子（如“温度超过上限”而非“温控不行”） |
| 大骨分类僵化，所有问题都5M1E | 根据实际问题调整，服务业可改用其他框架                       |
| 只有一个层级，未继续追问     | 持续问“为什么会发生这个？”，直到末端原因可以直接下手改善     |
| 把鱼骨图当最终答案，不验证   | 必须用数据或现场调查验证，否则就是猜测                       |
| 单人制作，缺少团队视角       | 至少包含与该过程直接相关的一线人员和跨职能专家               |

---

## 10. 结语

鱼骨图不是什么神秘的工具，而是一场**结构化的团队对话**。当问题出现时，它阻止你“凭直觉扑向第一个想到的原因”，而是用一张图描绘出因果的完整版图，再冷静地用事实去筛选真正的元凶。

下一次，当产线停摆、客诉涌来时，请不急着下令调机，而是拉过白板，画一条长长的骨线，写下那个清晰的问题，然后对团队说：**“来，我们一起找出它到底是怎么发生的。”**

那才是系统性分析问题的开始。
